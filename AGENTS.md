# AGENTS.md

## 环境

- conda 环境名：待确认
- 依赖入口：`requirements.txt`

## 修改前先读

- `README.md`
- `docs/notes.md`
- 当前需求涉及的代码文件
- 当前需求涉及的配置文件

## 禁止乱改

- 不改无关代码、算法逻辑、默认参数和数据路径。
- 不改 `data/` 下材料数据。
- 不删除历史配置或已有输出约定。
- 信息不足时直接写 `待确认`，不要编造。

## 修改规范

- 先分析，再改动。
- 只做与需求直接相关的最小修改。
- 改代码加必要注释。
- 改 YAML 时，新增项或重要修改要加注释。
- 说明受影响文件和验证方式；未验证就直接写未验证。

## 文档更新规则

- 每次实际修改后都要更新 `docs/notes.md` 和 `docs/logs/2026-03.md`。
- 如果命令、配置路径或输出路径变化，同步更新 `README.md`。

## Git 规范

- branch 命名建议：`feat/...`、`fix/...`、`docs/...`、`exp/...`
- commit message 建议：`feat: ...`、`fix: ...`、`docs: ...`

## 修改后输出格式

- 先写分析结论
- 再列修改文件
- 再写修改内容
- 再写验证
- 有待确认项就单列说明
